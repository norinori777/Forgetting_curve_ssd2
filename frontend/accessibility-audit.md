# Accessibility Audit (MVP)

対象: ホーム、カード一覧、学習カード登録画面（検索/フィルタ/選択/復習開始/バルク操作/削除確認モーダル/登録フォーム）

## キーボード操作

- ブランド領域: Tab で固定ヘッダー内のアイコン + サービス名リンクに到達でき、ホームへ戻れる
- トップレベルナビゲーション: Tab で固定ヘッダー内の「学習カード登録」「カード一覧」「復習」「統計」「設定」に到達できる
- ホーム:
  - 4 指標 summary は見出しと値の対になっており、主要導線より先に読み取れる
  - 「復習を始める」「カード一覧」「学習カード登録」「設定」に Tab だけで到達できる
  - 初回利用、今日の復習なし、取得失敗の各状態で、案内文と次アクションがテキストで認識できる
- 検索入力: Tabでフォーカス可能、入力可能
- 学習カード登録:
  - タイトル、学習内容、タグ入力は Tab で順に到達できる
  - 登録モード切替は `role="tablist"` / `role="tab"` として認識でき、Tab で到達後に Enter または Space で `単票登録` と `CSV一括登録` を切り替えられる
  - コレクション選択ボタンから単一選択モーダルを開ける
  - 登録、入力リセット、一覧へ戻るの主要操作にキーボードで到達できる
  - 必須エラーと送信エラーはテキストで認識できる
  - CSV一括登録モードでは、`CSVファイル` 入力、`一括登録する`、`キャンセル`、エラー一覧、取り込みプレビューの順で主要情報に到達できる
  - CSV検証エラーは `role="alert"` または `aria-label="CSVエラー一覧"` を通じてテキストで認識できる
  - 取り込みプレビューは件数サマリと行テーブルを含み、エラー行の状態テキストを視認できる
- 設定画面コレクション管理:
  - 設定カテゴリ、コレクション名入力、補足メモ、登録、一覧再読み込み、各行の編集/削除導線に Tab で到達できる
  - 編集モーダルと削除ダイアログは `role="dialog"` + `aria-modal="true"` で開き、Esc またはキャンセル操作で閉じられる
  - 必須エラー、重複エラー、読み込み失敗、削除不可理由、成功メッセージがテキストで認識できる
- ステータスフィルタ: 単一選択のセレクトとして Tab / Arrow で操作可能
- 共用フィルタモーダル: ボタンで開き、ラジオ切替・候補検索・チェックボックス複数選択をキーボードで操作可能
- 一覧の選択: 各カードのチェックボックスにTabで到達し、Spaceで切替可能
- バルク操作: 選択0件のとき操作ボタンは無効（disabled）
- ショートカット: Ctrl+F で検索、A で全選択トグル、R で復習開始、Delete で削除確認を開ける
- 削除確認モーダル:
  - `role="dialog"` + `aria-modal="true"`
  - Escでキャンセル可能
  - 初期フォーカスは「削除を確定」ボタン

## 目視確認ポイント

- `/` ではホーム見出し、`/cards` ではカード一覧見出しが表示され、固定ヘッダー/フッターは維持される
- `/` ではホーム見出し、4 指標 summary、主要導線、recent activities が表示される
- ホーム取得失敗時は `role="alert"` のエラー導線または再試行 UI で認識できる
- `/cards/create` では学習カード登録見出し、fixed compact header とフッターが維持される
- 共通ヘッダーに breadcrumb 専用領域が存在せず、ブランド領域クリックでホームへ戻れる
- `/cards/create` の CSV モードでは、形式ガイド、文字コード案内、選択ファイル名、エラー一覧、取り込みプレビューが 1 画面内で確認できる
- `/settings` では設定見出し、コレクション管理セクション、登録フォーム、登録済み一覧、モーダル/ダイアログが共通レイアウト内に表示される
- エラー表示が `role="alert"` で通知される
- 空状態で条件リセットができる
- ソート UI が一覧ヘッダ内に埋め込まれ、現在値が視認できる
- タグ/コレクションの選択結果がボタン横サマリに反映される
- 登録画面では入力プレビューが現在値を反映し、未保存注意が dirty state で表示される
- 設定画面では空状態、登録成功、更新成功、削除成功、重複エラー、削除不可状態が視認できる
