# Feature Specification: カード回答項目

**Feature Branch**: `001-card-answer-field`  
**Created**: 2026-03-22  
**Status**: Draft  
**Input**: User description: "カードに学習内容に対する回答を登録できる項目を追加する。回答は任意項目とする。理由としては、忘却曲線は、学習サイクルが目的であり、テストではなく学習する場合、テストとならないこともあるので、回答は任意とする。一覧の回答は、通常非表示としており、回答表示リンクをクリックすると、リンクから回答に置き換える仕様とする。設定にて、リンクとするか、一覧にそのまま回答を表示するかは変更可能とする。ここでは、設定値を参照して、切り替えるところまでを実装して、設定画面については、別のタイミングで実装することとする。"

## Clarifications

### Session 2026-03-22

- Q: 回答項目の入力形式は何にする？ → A: 複数行のプレーンテキストを許可する
- Q: 回答はどの入力導線で登録する？ → A: 今回はデータモデルと一覧表示だけを対象にし、回答の登録 UI は含めない
- Q: 一覧で長い回答はどう表示する？ → A: 一覧では回答を最大数行まで表示し、超過分は省略表示にする
- Q: 一覧検索は回答も対象に含める？ → A: 既存検索に回答も含める

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 一覧で必要なときだけ回答を見る (Priority: P1)

学習者は、カード一覧を見ながら、必要なカードだけ回答を確認したい。通常は回答を隠して一覧性を保ち、必要な瞬間だけ回答を表示したい。

**Why this priority**: 学習中の視線移動と情報量を抑えながら、必要なときだけ答え合わせできることが、この機能の中心価値だから。

**Independent Test**: 回答を持つカードが一覧に表示されている状態で、回答表示リンクを押したカードだけ回答が表示され、他のカード表示は変わらないことを確認する。

**Acceptance Scenarios**:

1. **Given** 回答を持つカードが一覧に表示されており、回答表示方法がリンク表示に設定されている, **When** ユーザがそのカードの回答表示リンクを押す, **Then** そのカードではリンクが回答に置き換わって表示される
2. **Given** 複数のカードが一覧に表示されている, **When** ユーザが1件のカードだけで回答表示リンクを押す, **Then** そのカード以外の表示状態は変わらない
3. **Given** 回答を持たないカードが一覧に表示されている, **When** ユーザが一覧を見る, **Then** 回答が未登録であることが分かり、回答表示リンクは表示されない

---

### User Story 2 - 回答がなくてもカードを使い続ける (Priority: P1)

学習者は、暗記ではなく理解や反復を目的とするカードにも対応したいため、回答が登録されていないカードでも通常どおり使い続けたい。

**Why this priority**: 回答を必須にすると、学習サイクル重視の使い方を阻害し、この機能の前提に反するから。

**Independent Test**: 回答ありと回答なしの両方のカードデータを一覧で読み出し、どちらも有効なカードとして扱われることを確認する。

**Acceptance Scenarios**:

1. **Given** 回答を持たないカードデータが存在する, **When** ユーザが一覧や通常の学習導線でそのカードを利用する, **Then** 回答がないことだけを理由にカードが除外されたり利用不能になったりしない
2. **Given** 回答を持つカードデータが存在する, **When** ユーザが一覧を表示する, **Then** 回答の有無を含むカードデータが欠落せずに読み出される
3. **Given** 回答ありと回答なしのカードが混在している, **When** ユーザが一覧を利用する, **Then** 回答の有無に応じた表示だけが変わり、カード自体の利用可否は変わらない
4. **Given** 回答内に検索語を含むカードが存在する, **When** ユーザが一覧検索を実行する, **Then** 回答内の一致によっても対象カードが検索結果に含まれる

---

### User Story 3 - 設定値に応じた既定表示で学習する (Priority: P2)

学習者は、自分の学習スタイルに合わせて、一覧で回答をリンク表示にするか、そのまま表示するかを切り替えたい。今回は設定画面そのものではなく、既存の設定値に応じて一覧表示が切り替わることを求める。

**Why this priority**: 一覧の情報量に対する好みは個人差が大きく、設定値を尊重した表示切替が継続利用のしやすさにつながるから。

**Independent Test**: 回答表示方法の設定値をそれぞれ切り替えた状態で一覧を表示し、リンク表示と常時表示が既定表示として反映されることを確認する。

**Acceptance Scenarios**:

1. **Given** 回答表示方法がリンク表示に設定されている, **When** ユーザがカード一覧を開く, **Then** 回答を持つカードは回答表示リンクを既定表示とする
2. **Given** 回答表示方法が常時表示に設定されている, **When** ユーザがカード一覧を開く, **Then** 回答を持つカードは回答本文を既定表示とする
3. **Given** 回答表示方法の設定値が変更されている, **When** ユーザが次回一覧を表示する, **Then** 一覧は変更後の設定値に従って回答表示方法を切り替える

---

### Edge Cases

- 回答が空欄のカードは、回答を持つカードと誤認されないように表示されること
- 回答が長文や複数行でも、一覧では最大数行まで表示し、超過分は省略表示して一覧性を保つこと
- 一覧表示中にデータを再読み込みした場合、回答の既定表示は現在の設定値に基づいて再判定されること
- 回答表示方法の設定値が取得できない、または不正な場合でも、一覧表示が破綻せず安全な既定表示にフォールバックすること

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST カードに対して単一の回答項目を保持できる
- **FR-001a**: System MUST 回答項目では複数行のプレーンテキストを保持できる
- **FR-002**: System MUST 回答を未入力のままでもカードを有効なカードとして保持および利用できる
- **FR-003**: System MUST 回答が未登録の既存カードを有効なカードとして扱い続ける
- **FR-004**: System MUST 回答が登録されているカードについて、回答表示方法がリンク表示のときは一覧上で回答本文を初期状態では非表示にする
- **FR-005**: System MUST 回答表示方法がリンク表示のときは、回答を持つ各カードに対して回答表示リンクを表示する
- **FR-006**: System MUST ユーザが回答表示リンクを実行したとき、そのカード内でリンク表示を回答本文に置き換える
- **FR-007**: System MUST 1件のカードで回答を表示しても、他のカードの回答表示状態を変更しない
- **FR-008**: System MUST 回答表示方法が常時表示のときは、回答を持つカードの一覧項目内に回答本文を既定表示する
- **FR-008a**: System MUST 一覧に表示する回答本文は最大数行までとし、超過分は省略表示して一覧性を維持する
- **FR-009**: System MUST 回答が未登録のカードについて、回答が存在しないことを分かる形で示し、回答表示リンクは提供しない
- **FR-010**: System MUST 一覧での回答表示方法を、現在の設定値を参照して決定する
- **FR-011**: System MUST 一覧を新たに表示したとき、または一覧データを再取得したとき、現在の設定値に従って回答の既定表示を適用する
- **FR-012**: System MUST 回答表示方法の設定値を参照できない場合、回答本文を初期状態では非表示にする安全な既定表示で一覧を継続表示する
- **FR-013**: System MUST 今回の機能範囲では、回答表示方法の設定画面自体を変更せず、既存の設定値参照の反映までを対象とする
- **FR-014**: System MUST 今回の機能範囲では、回答を登録または編集するための新しい UI を追加しない
- **FR-015**: System MUST 既存の一覧検索では、タイトルや問題文に加えて回答も検索対象に含める

### Key Entities *(include if feature involves data)*

- **Card**: 学習内容と任意の回答を持つ学習単位。回答の有無にかかわらず利用できる
- **Answer**: カードに紐づく任意入力の回答内容。複数行のプレーンテキストを扱い、登録されている場合のみ一覧で表示対象となる
- **Answer Display Preference**: 一覧で回答を初期状態でリンク表示にするか、本文表示にするかを表す利用者設定

### Assumptions

- 回答はカードごとに 1 つ保持する
- 回答は複数行のプレーンテキストとして扱い、装飾記法は解釈しない
- 回答は任意入力であり、未入力を正常値として扱う
- 回答表示方法の設定値は、今回の機能着手時点で別経路から参照可能である
- 回答表示方法の設定値が参照できない場合の既定表示は、回答を隠して必要時のみ表示する方式とする
- 設定画面で回答表示方法を変更する UI は今回の範囲に含めない
- 回答を登録または編集する UI は別 feature で扱い、今回は回答データを読み出して一覧表示へ反映する
- 一覧検索は既存の検索導線を拡張して、回答テキストも対象に含める

### Scope Boundaries

- In Scope: カードへの任意回答を扱えるデータモデル、回答なしカードの継続利用、一覧での回答の既定非表示と個別表示、設定値に応じた一覧表示切替、回答を含む一覧検索
- Out of Scope: 回答を登録または編集する UI の提供、回答表示方法を変更する設定画面の提供、回答を使った採点ロジックや学習アルゴリズムの変更

### Dependencies

- カード情報の保存先が、任意の回答項目を保持し一覧取得時に返却できること
- 一覧画面が、利用者ごとの回答表示方法の設定値を取得できること
- 一覧検索のデータ取得が、回答テキストを検索対象に含められること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 回答ありと回答なしの両方のカードが、検証対象ケースの 100% で一覧取得および再表示に成功する
- **SC-002**: 回答表示方法がリンク表示のとき、ユーザは各カードの回答を 1 回の操作で表示できる
- **SC-003**: 回答表示方法が常時表示のとき、回答を持つカードは検証対象ケースの 100% で一覧初期表示時から回答本文を確認できる
- **SC-003a**: 長文回答を持つカードでも、検証対象ケースの 100% で一覧レイアウトを崩さずに省略表示される
- **SC-004**: 回答表示方法の設定値を変更した後、次回一覧表示時の既定表示が検証対象ケースの 100% で設定値と一致する
- **SC-005**: ユーザテストまたは受け入れ確認で、参加者の 90% 以上が「回答あり」と「回答なし」を一覧上で正しく判別できる
- **SC-006**: 回答内にのみ検索語を含むカードが、検証対象ケースの 100% で一覧検索結果に含まれる
