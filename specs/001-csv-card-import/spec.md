# Feature Specification: 学習カードCSV一括登録

**Feature Branch**: `001-csv-card-import`  
**Created**: 2026-03-28  
**Status**: Draft  
**Input**: User description: "CSVファイルをアップロードすることで、そのCSVファイルから学習カードの一括登録機能を学習登録画面に追加してください。画面イメージのascii uiを作成してください。CSVファイルの形式は、以下とする。第1カラム（必須）：タイトル、第2カラム（必須）：学習内容（改行は、\nで表現）、第3カラム（任意）：回答、第4カラム（任意）：タグ、第5カラム（任意）：コレクション。CSVファイルの以下の形式チェックを実施して、エラーの場合は、エラーメッセージを表示すること。任意がない場合は、エラーとする。ないコレクションが指定された場合は、エラーとする。"

## Clarifications

### Session 2026-03-28

- Q: 一括登録中に一部の行だけ保存失敗した場合、登録結果はどう扱うべきか？ → A: 1件でも保存失敗が起きたら、今回のCSV登録は全件ロールバックして未登録にする
- Q: CSVのタグ列で複数タグを表現する場合、どの区切り文字を使うべきか？ → A: タグ列内の複数タグはセミコロン `;` 区切りで表現する
- Q: CSVのタグ列に未登録タグが含まれていた場合、どう扱うべきか？ → A: CSVのタグ列に未登録タグがあれば、新規作成してカードに紐づける
- Q: CSVファイルの文字コードはどこまで受け付けるべきか？ → A: UTF-8 / UTF-8 BOM / Shift_JIS を受け付ける
- Q: CSVの先頭行が見出し行である場合、どう扱うべきか？ → A: 先頭行が「タイトル,学習内容,回答,タグ,コレクション」と一致する場合のみヘッダー行として読み飛ばす

### Session 2026-03-29

- Q: 追加する回答列は必須か任意か？ → A: 回答列は任意とし、空欄でも一括登録できる

## User Scenarios & Testing *(mandatory)*

### User Story 1 - CSVからカードをまとめて登録する (Priority: P1)

ユーザは、学習カード登録画面でCSVファイルをアップロードし、複数の学習カードをまとめて登録したい。

**Why this priority**: 複数カードを手入力せずに取り込めることが本機能の中心価値であり、登録作業時間を大きく削減できるためである。

**Independent Test**: 学習カード登録画面で仕様通りのCSVを選択し、内容確認後に一括登録を実行して、CSV内の全カードが登録されることを確認する。

**Acceptance Scenarios**:

1. **Given** ユーザが学習カード登録画面を開いている, **When** 指定形式のCSVファイルを選択して取り込みを実行する, **Then** CSV内の各行がカード候補として認識される
2. **Given** CSV内の各行にタイトルと学習内容が含まれている, **When** ユーザが一括登録を確定する, **Then** 各行から学習カードが作成される
3. **Given** 学習内容欄に `\n` が含まれている, **When** 取り込み結果が生成される, **Then** 利用者が意図した改行を含む学習内容として扱われる
4. **Given** 回答欄が空欄の行を含むCSVを取り込む, **When** 一括登録を確定する, **Then** その行は回答未設定のカードとして登録される
5. **Given** タグ欄が空欄の行を含むCSVを取り込む, **When** 一括登録を確定する, **Then** その行はタグ未設定のカードとして登録される
6. **Given** コレクション欄が空欄の行を含むCSVを取り込む, **When** 一括登録を確定する, **Then** その行はコレクション未設定のカードとして登録される
7. **Given** 一括登録中に1件でも保存失敗が発生した, **When** システムが登録処理を完了する, **Then** 今回のCSV登録結果は全件未登録のままとなり、部分的な登録結果を残さない
8. **Given** 第4カラムに複数タグが記載されている, **When** 取り込み結果を解釈する, **Then** セミコロン `;` 区切りで複数タグとして認識する
9. **Given** 第4カラムに未登録タグが含まれている, **When** 一括登録を確定する, **Then** 未登録タグは新規作成されたうえでカードに紐づく
10. **Given** CSVの先頭行が「タイトル,学習内容,回答,タグ,コレクション」と一致している, **When** 取り込み結果を生成する, **Then** 先頭行はヘッダーとして読み飛ばされ、登録対象件数に含めない

---

### User Story 2 - 形式エラーを事前に把握する (Priority: P2)

ユーザは、不正なCSVを取り込んだ場合に、どこに問題があるかを画面上で把握し、修正後に再アップロードしたい。

**Why this priority**: 一括登録では1件でも形式を誤ると大量データが失敗しやすく、失敗理由が分からないと再作業コストが高くなるためである。

**Independent Test**: 不正なCSVを取り込み、エラー内容が表示され、登録処理が止まることを確認する。

**Acceptance Scenarios**:

1. **Given** 第3カラムから第5カラムまでのいずれかが存在しないCSVを選択した, **When** 形式チェックを実行する, **Then** 必須の列構成を満たしていないエラーが表示され、登録へ進めない
2. **Given** タイトルまたは学習内容が空欄の行を含むCSVを選択した, **When** 形式チェックを実行する, **Then** 該当行の必須項目不足が表示され、登録へ進めない
3. **Given** 存在しないコレクション名を含むCSVを選択した, **When** 形式チェックを実行する, **Then** 該当行と指定コレクション名が分かるエラーが表示され、登録へ進めない
4. **Given** 取り込みに失敗したCSVがある, **When** ユーザがエラー表示を確認する, **Then** ファイルを修正して再アップロードすべき内容を把握できる

---

### User Story 3 - 取り込み前に登録対象を確認する (Priority: P3)

ユーザは、アップロード直後に取り込み対象件数や内容の概要を確認してから、一括登録を確定したい。

**Why this priority**: 一括登録は影響件数が大きいため、実行前に内容を確認できることが誤登録抑止に直結するためである。

**Independent Test**: 正常CSVを取り込み、登録予定件数とサンプル内容が確認でき、キャンセルまたは再選択が行えることを確認する。

**Acceptance Scenarios**:

1. **Given** 正常なCSVを取り込んだ, **When** 画面が検証結果を表示する, **Then** 登録予定件数と主要項目の一覧または要約を確認できる
2. **Given** 正常なCSVを取り込んだ, **When** ユーザが別のCSVを再選択する, **Then** 直前の取り込み結果は置き換わる
3. **Given** 正常なCSVを取り込んだ, **When** ユーザが一括登録をキャンセルする, **Then** 登録は実行されず、他の登録操作へ戻れる

### Edge Cases

- CSVファイル自体が未選択のまま取り込み操作を行った場合は、ファイル選択を促して登録へ進めない
- CSVの列数が行ごとに不一致な場合は、形式不正として扱う
- 第3カラムから第5カラムの値が空欄でも、列自体が存在していれば形式エラーにはしない
- 第4カラムに値がある場合は、セミコロン `;` 区切りで複数タグを表現する
- 学習内容内の `\n` は改行として解釈し、それ以外の列区切りや行区切りを壊さない
- UTF-8、UTF-8 BOM、Shift_JIS 以外の文字コードは形式不正として扱う
- 先頭行が「タイトル,学習内容,回答,タグ,コレクション」と完全一致する場合のみヘッダー行として読み飛ばし、それ以外は1行目からデータ行として扱う
- 同一CSV内に同名タイトルの行が複数あっても、重複禁止要件が定義されていない限り別カードとして扱う
- 第4カラムに未登録タグが含まれていても形式エラーにはせず、新規タグとして扱う
- 存在しないコレクション名が1件でも含まれる場合は、ユーザが全エラーを解消するまで一括登録を完了しない
- 形式エラーがあるCSVを取り込んだ後に正常CSVへ差し替えた場合は、前回エラー表示を引き継がない
- 非常に大きいCSVでも、少なくともユーザが件数とエラー件数を把握できる要約を表示する
- 一括登録実行後に保存失敗が発生した場合は、成功した行だけを残さず全件未登録に戻す

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST 学習カード登録画面に、単票登録と区別して使えるCSV一括登録の導線または領域を表示する
- **FR-002**: System MUST ユーザがCSVファイルを選択または差し替えできるようにする
- **FR-003**: System MUST CSV取り込み前に、受け付ける列順を画面上で案内する
- **FR-003a**: System MUST 受け付ける文字コードが UTF-8、UTF-8 BOM、Shift_JIS であることを画面上で案内する
- **FR-004**: System MUST CSVの第1カラムをタイトル、第2カラムを学習内容、第3カラムを回答、第4カラムをタグ、第5カラムをコレクションとして扱う
- **FR-004a**: System MUST 先頭行が「タイトル,学習内容,回答,タグ,コレクション」と一致する場合のみ、その先頭行をヘッダーとして読み飛ばす
- **FR-005**: System MUST 第1カラムと第2カラムを必須値として検証する
- **FR-005a**: System MUST 第3カラムの回答について、値は空欄を許容する
- **FR-006**: System MUST 第3カラムから第5カラムについて、値は空欄を許容しても列自体の欠落は形式エラーとして扱う
- **FR-007**: System MUST 学習内容内の `\n` を改行表現として解釈する
- **FR-008**: System MUST 回答欄が空欄の行を回答未設定として扱う
- **FR-009**: System MUST タグ欄が空欄の行をタグ未設定として扱う
- **FR-009a**: System MUST 第4カラムに値がある場合、セミコロン `;` 区切りで複数タグを解釈できるようにする
- **FR-009b**: System MUST 第4カラムに未登録タグが含まれている場合、そのタグを新規作成してカードに紐づける
- **FR-010**: System MUST コレクション欄が空欄の行をコレクション未設定として扱う
- **FR-011**: System MUST コレクション欄に既存コレクションと一致しない値が含まれる場合、該当行を特定できるエラーを表示する
- **FR-012**: System MUST CSVの列構成、必須値不足、存在しないコレクションを検証し、エラーが1件でもある場合は一括登録を実行しない
- **FR-012a**: System MUST CSVの文字コードを検証し、UTF-8、UTF-8 BOM、Shift_JIS のいずれにも該当しない場合はエラーを表示して一括登録を実行しない
- **FR-013**: System MUST 形式エラー表示に、少なくとも問題種別、該当行、ユーザが修正すべき内容を含める
- **FR-014**: System MUST 正常CSVを受け付けた後、一括登録前に登録予定件数と取り込み対象の概要を確認できるようにする
- **FR-015**: System MUST ユーザが一括登録を確定した場合、CSVの各正常行から学習カードをまとめて登録する
- **FR-015a**: System MUST 回答列に値がある場合、その値を学習カードの回答として登録する
- **FR-015b**: System MUST 一括登録中に1件でも保存失敗が発生した場合、今回のCSVで処理した全行の登録結果をロールバックし、部分成功を残さない
- **FR-016**: System MUST 一括登録成功後、何件登録できたかが分かる完了メッセージを表示する
- **FR-017**: System MUST ユーザが取り込み結果を破棄して別ファイルを再選択できるようにする
- **FR-018**: System MUST 学習カード登録画面において、単票登録の操作とCSV一括登録の操作を視覚的に区別し、誤って別操作を実行しにくい構成にする
- **FR-019**: System MUST PC表示とモバイル表示の双方で、CSV形式案内、ファイル選択、検証結果、実行操作の順に把握できる画面構成を提供する
- **FR-020**: System MUST 画面イメージとして定義する ASCII UI と整合する情報配置を提供する

### Key Entities *(include if feature involves data)*

- **CSV Import File**: 学習カードを複数行で表現するアップロード対象。各行はタイトル、学習内容、回答、タグ、コレクションの順で構成される
- **CSV Import Row**: 1件の学習カード候補を表す行。行番号、列値、検証結果を持つ
- **Import Validation Result**: 取り込み可否の判定結果。問題種別、対象行、修正案内を含む
- **Bulk Registration Summary**: 正常件数、エラー件数、登録予定件数など、ユーザが実行前後に確認する要約情報
- **Learning Card**: タイトル、学習内容、タグ、コレクションを持つ登録対象

### Assumptions

- 先頭行が「タイトル,学習内容,回答,タグ,コレクション」と一致する場合のみヘッダー行として扱い、それ以外は1行目からデータ行として扱う
- 受け付ける文字コードは UTF-8、UTF-8 BOM、Shift_JIS とする
- 「任意」は第3カラムから第5カラムの値が空欄でもよいことを意味し、列そのものが欠落したCSVは形式不正として扱う
- 回答列は空欄を許容し、空欄時は回答未設定として扱う
- タグ列に値がある場合、1セル内の複数タグはセミコロン `;` 区切りで表現する
- タグ列に未登録タグが含まれている場合でも、一括登録時に新規タグとして扱えるものとする
- 一括登録対象は新規カード作成のみであり、既存カード更新は対象外とする
- 一括登録成功後の完了案内は、登録件数が分かるメッセージを表示する
- 存在するコレクションの判定は、ユーザが現在選択可能なコレクション集合を基準とする
- 一括登録処理は全件成功時のみ確定し、1件でも保存失敗した場合は今回のCSV分を未登録状態へ戻す

### Scope Boundaries

- In Scope: 学習カード登録画面へのCSV一括登録導線追加、CSV形式案内、ファイル選択、形式チェック、エラー表示、取り込み結果要約、複数カードの新規登録、ascii ui 作成
- Out of Scope: CSVエクスポート、既存カード更新、コレクション新規作成、重複カードの自動マージ、アップロード履歴管理

### Dependencies

- 学習カード登録画面が既に存在し、CSV一括登録領域を追加できること
- 既存コレクション一覧を参照して照合できること
- 複数の学習カードを連続登録できる永続化手段が存在すること
- 取り込み結果の成功件数と失敗理由をユーザに表示できること

### Screen Reference

画面構成の基準は [ascii_ui.txt](./ascii_ui.txt) とする。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 利用者は 100 件までの標準的なCSVを 3 分以内に取り込み開始から登録完了まで進められる
- **SC-002**: 正常なCSVを使った一括登録で、登録対象行の 100% が欠落なくカードとして反映される
- **SC-003**: 形式不正を含むCSVでは、利用者が修正対象の行と理由を初回表示で把握できる
- **SC-004**: 利用者テストまたは受け入れ確認において、主要操作である「アップロードして一括登録する」を 90% 以上が初回で完了できる
