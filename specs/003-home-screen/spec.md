## User Scenarios & Testing *(mandatory)*

### User Story 1 - 今日やることをひと目で把握する (Priority: P1)

ユーザは、アプリを開いた直後に今日やるべき学習内容を理解したい。

**Why this priority**: ホーム画面は最初の接点であり、ここで迷うと復習開始や次の操作に進みにくくなるため。

**Independent Test**: ホーム画面を表示したときに、今日の復習件数、学習状況、主要な次の行動が 1 画面内で認識できることを確認する。

**Acceptance Scenarios**:

1. **Given** ユーザに今日の復習対象がある, **When** ホーム画面を開く, **Then** 今日の復習対象数、期限超過数、未学習数、連続学習日数の 4 指標が分かる
2. **Given** ホーム画面が表示されている, **When** ユーザが画面上部を見る, **Then** 学習状況の要約と主要な行動導線を同時に認識できる
3. **Given** ホーム画面が表示されている, **When** ユーザが最初に次の操作を探す, **Then** 復習開始、カード一覧、学習カード登録、設定の導線がすぐ見つかる
4. **Given** 今日の復習対象が存在する, **When** ユーザが「復習を始める」を選ぶ, **Then** 今日の復習対象だけを開始対象として復習へ進む

---

### User Story 2 - 主要な操作へ迷わず移動する (Priority: P2)

ユーザは、ホーム画面から復習、カード一覧、カード登録などの主要な行動へ迷わず移動したい。

**Why this priority**: ホーム画面が情報表示だけで終わると価値が低く、実際の作業へつながる導線が必要なため。

**Independent Test**: ホーム画面上の主要導線を見て、どこから何ができるか説明できることを確認する。

**Acceptance Scenarios**:

1. **Given** ホーム画面が表示されている, **When** ユーザが主要操作を見る, **Then** 復習開始、カード一覧、カード登録、設定への導線を区別できる
2. **Given** ホーム画面が表示されている, **When** ユーザが最近のアクティビティを見る, **Then** 主要イベントが直近 3 件まで表示され、次にどの画面へ進むべきか判断できる

---

### User Story 3 - 状態に応じた案内を受ける (Priority: P3)

ユーザは、データがない場合や今日の復習がない場合でも、次に何をすべきか分かる案内を受けたい。

**Why this priority**: 初回利用や復習対象なしの状態で案内が弱いと、ホーム画面が空白に見えて利用価値が下がるため。

**Independent Test**: 初回利用、復習対象なし、取得失敗の各状態で、適切な案内文と行動導線があることを確認する。

**Acceptance Scenarios**:

1. **Given** 学習カードが 0 件である, **When** ホーム画面を開く, **Then** 初回利用向けの説明とカード登録導線が表示される
2. **Given** 今日の復習対象が存在しない, **When** ホーム画面を開く, **Then** 代替行動を示す案内が表示される
3. **Given** ホーム画面の情報取得に失敗する, **When** ホーム画面を開く, **Then** 失敗が分かり、再試行の導線が表示される

### Edge Cases

- 今日の復習対象が 0 件でも、画面が空白にならず、代替行動が示される
- 学習カードが 0 件でも、最初の登録導線がすぐ見つかる
- 最近のアクティビティが存在しない場合でも、レイアウトが崩れない
- 集計値の一部がない場合でも、ユーザが理解できる表示を維持する
- モバイル表示でも主要導線が見切れず、縦方向に順にたどれる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST 初期表示ルート `/` にホーム画面を提供する
- **FR-002**: System MUST ホーム画面で今日の復習対象数、期限超過数、未学習数を分かる形で表示する
- **FR-003**: System MUST ホーム画面で連続学習日数を表示し、学習サマリを「今日の復習対象数」「期限超過数」「未学習数」「連続学習日数」の 4 指標に固定する
- **FR-004**: System MUST ホーム画面で、復習開始、カード一覧、学習カード登録、設定への主要導線を表示する
- **FR-004a**: System MUST ホーム画面の「復習を始める」を、今日の復習対象だけを開始する導線として提供する
- **FR-005**: System MUST ホーム画面の情報を、学習サマリ、主要導線、最近のアクティビティを中心に構成する
- **FR-006**: System MUST ホーム画面で最近のアクティビティを主要イベントの直近 3 件まで表示する
- **FR-007**: System MUST 初回利用、復習対象なし、取得失敗の各状態に応じた表示を提供する
- **FR-007a**: System MUST 学習カードが 0 件のとき、初回利用状態を表示する
- **FR-008**: System MUST ホーム画面の情報配置と主要 UI 構成を ascii_ui.txt に整合させる
- **FR-009**: System MUST デスクトップとモバイルの双方で主要情報と主要導線を認識できる構成にする
- **FR-010**: System MUST キーボード操作のみでも主要導線へ到達できる
- **FR-011**: System MUST ホーム画面のデザインは既存トップレベルナビゲーションと視覚的に整合する
- **FR-012**: System MUST ホーム画面はカード一覧の詳細仕様と混在せず、独立した feature として管理される

### Key Entities *(include if feature involves data)*

- **Home Summary**: 今日の復習件数、期限超過件数、未学習件数、直近の学習状況をまとめた表示単位
- **Primary Action**: 復習開始、カード一覧、学習カード登録、設定など、ホーム画面から直接選べる主要導線
- **Activity Summary**: 最近行った学習やカード操作の簡易履歴
- **Home State**: 通常表示、初回利用、復習対象なし、取得失敗などの画面状態

### Assumptions

- ホーム画面は既存トップレベルナビゲーションの一部として表示される
- 復習開始、カード一覧、学習カード登録、設定の各導線は既存または別 feature の画面へ接続される
- ホーム画面の「復習を始める」は今日の復習対象だけを開始する
- 最近のアクティビティは詳細履歴ではなく、主要イベントの直近 3 件を要約表示する
- 初回利用状態は学習カードが 0 件のときに表示する
- 優先プレビュー枠は設けず、ホーム画面は学習サマリ、主要導線、最近のアクティビティに情報を絞る
- 学習サマリの固定項目は、今日の復習対象数、期限超過数、未学習数、連続学習日数の 4 指標とする
- ホーム画面に表示する数値や活動履歴は、入口画面として必要な範囲に要約される
- 詳細な一覧操作や個別カード編集はホーム画面では扱わない

### Scope Boundaries

- In Scope: ホーム画面の ASCII UI、主要導線、学習サマリ、最近のアクティビティ、初回利用/復習対象なし/取得失敗の状態設計
- Out of Scope: カード一覧の検索・フィルタ・ソート詳細、復習画面本体、統計画面本体、設定画面本体、バックエンド実装詳細

### Dependencies

- トップレベルナビゲーションと `/` ルートが存在すること
- 復習開始、カード一覧、学習カード登録、設定への遷移先が別途用意されること
- ホーム画面の見た目を合わせるための共通デザイントークンとレイアウト方針が維持されること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザがホーム画面を開いてから主要導線を見つけるまでが 5 秒以内である
- **SC-002**: ユーザの 90% 以上が、ホーム画面だけを見て次に取るべき行動を説明できる
- **SC-003**: デスクトップとモバイルの両方で、主要導線が初回表示範囲または 1 回の短いスクロールで到達できる
- **SC-004**: 初回利用、復習対象なし、取得失敗の各状態で、ユーザが次の行動を判断できる表示が維持される
