## Clarifications

### Session 2026-03-22

- Q: この feature における「検索結果」は何を指すか？ → A: 開始時点の検索条件に一致する全カードを検索結果とみなす
- Q: 開始直前に一部カードが利用不能になった場合、復習開始をどう扱うか？ → A: 利用可能なカードだけで開始し、除外された件数と理由を表示する
- Q: 検索条件に一致するカード数が多すぎる場合、復習開始をどう扱うか？ → A: 上限件数までを対象に開始し、除外された件数と理由を表示する
- Q: 復習開始上限件数はいくつにするか？ → A: 200件

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 検索結果から復習を始める (Priority: P1)

学習者として、学習カード一覧で検索して見つかったカード群をそのまま復習対象にして開始したい。これにより、見つけたカードを探し直さずにすぐ復習へ移れる。

**Why this priority**: 一覧で探す操作と復習を始める操作がつながらないと、この機能の中心価値である「検索結果からそのまま復習」が成立しない。

**Independent Test**: 学習カード一覧で検索条件を指定して結果を表示し、その状態で「復習開始」を実行したとき、開始時点の検索条件に一致する全カードを対象に復習へ遷移できれば独立して価値を検証できる。

**Acceptance Scenarios**:

1. **Given** 学習カード一覧で検索条件を指定し、複数のカードが結果として表示されている, **When** 学習者が「復習開始」を実行する, **Then** 開始時点の検索条件に一致する全カードを対象に復習が開始される
2. **Given** 学習カード一覧で検索条件を指定し、表示中の検索結果が復習対象として妥当である, **When** 学習者が復習を開始する, **Then** 復習開始後に学習カード一覧ではなく復習画面へ遷移する
3. **Given** 学習カード一覧で検索条件を変更して検索結果が変わった, **When** 学習者がその状態で「復習開始」を実行する, **Then** 変更後の最新の検索条件に一致する全カードが復習対象になる
4. **Given** 学習カード一覧の検索条件に一致するカード数が 200 件を超えている, **When** 学習者が「復習開始」を実行する, **Then** 200 件までのカードで復習が開始され、除外された件数と理由が表示される

---

### User Story 2 - 検索結果のカードを順番に復習する (Priority: P1)

学習者として、検索結果に含まれていたカードを復習画面で 1 枚ずつ確認しながら復習したい。これにより、一覧で絞り込んだ意図を保ったまま学習を続けられる。

**Why this priority**: 復習開始だけ成功しても、復習画面で別のカードが混ざったり検索結果の一部しか表示されなければ、機能の信頼性が失われる。

**Independent Test**: 検索結果に 2 枚以上のカードがある状態から復習を開始し、復習画面でそのカード群だけが 1 枚ずつ表示されることを確認できれば独立して価値を検証できる。

**Acceptance Scenarios**:

1. **Given** 検索結果に 2 枚以上のカードが含まれている状態で復習が開始された, **When** 学習者が復習画面を開く, **Then** 検索結果に含まれていたカードのうち 1 枚目が表示される
2. **Given** 検索結果を対象とする復習が進行中である, **When** 学習者が次のカードへ進む, **Then** 同じ検索結果に含まれていた次のカードが表示される
3. **Given** 検索結果を対象とする復習が進行中である, **When** 学習者が最後の対象カードまで復習する, **Then** 検索結果に含まれていないカードは表示されず、対象カードの復習完了後に完了状態へ進める

---

### User Story 3 - 対象がないときに誤開始しない (Priority: P2)

学習者として、検索結果が空のときは復習を始められないか、始められない理由を理解したい。これにより、空の条件で誤って復習画面へ進んで混乱しない。

**Why this priority**: 対象がない状態の扱いが曖昧だと、開始失敗なのか仕様なのか分からず操作が止まる。

**Independent Test**: 検索条件に一致するカードが 0 件の状態を作り、復習開始の可否と表示される説明を確認できれば独立して価値を検証できる。

**Acceptance Scenarios**:

1. **Given** 学習カード一覧の検索結果が 0 件である, **When** 学習者が復習開始可能な状態を確認する, **Then** 復習対象が存在しないことが分かる
2. **Given** 学習カード一覧の検索結果が 0 件である, **When** 学習者が復習開始を試みる, **Then** 空の検索結果のまま復習が進行せず、条件見直しまたは一覧へ戻るための導線が提示される
3. **Given** 学習カード一覧の検索結果に含まれる一部カードが開始直前に利用不能になった, **When** 学習者が復習開始を実行する, **Then** 利用可能なカードだけで復習が開始され、除外された件数と理由が表示される
4. **Given** 学習カード一覧の検索結果件数が 200 件を超えている, **When** 学習者が復習開始を実行する, **Then** 空の復習にはならず 200 件までで開始され、超過分の除外件数と理由が表示される

### Edge Cases

- 検索結果が 1 件だけの場合でも、その 1 件を対象に復習を開始して完了できること
- 検索条件を変えた直後に復習を開始した場合でも、開始時点の検索条件に一致する全カードが対象になること
- 復習開始後に一覧側の検索条件や表示内容が変わっても、進行中の復習対象は開始時点の対象集合を維持すること
- 検索結果に含まれていたカードの一部が開始直前に利用不能になった場合でも、利用可能なカードだけで復習を開始し、除外された件数と理由を利用者に分かる形で示すこと
- 検索結果件数が 200 件を超える場合でも、200 件までで復習を開始し、超過分が上限超過により除外されたことを利用者に分かる形で示すこと
- 復習対象が 0 件のまま開始しようとした場合でも、空の復習画面に遷移して操作不能にならないこと

## Assumptions

- ここでいう検索結果とは、学習カード一覧で開始時点の検索条件に一致する全カード集合を指す
- 検索結果には、検索語だけでなく一覧画面上で同時に適用されている絞り込み条件も含まれる
- 復習開始時の対象集合は開始時点で確定し、進行中に一覧画面側の表示が変化しても自動では入れ替わらない
- 開始直前に一部カードが利用不能になった場合は、そのカードを対象集合から除外したうえで、利用可能なカードが 1 件以上あれば復習を開始する
- 開始時点の検索結果件数が 200 件を超える場合は、200 件までを対象集合として確定し、超過分を上限超過として除外する
- 復習画面には、開始された対象集合を 1 枚ずつ順番に表示する既存の復習体験がある
- 検索結果内の表示順がある場合、復習画面でもその順序に沿って提示される

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、学習カード一覧で開始時点の検索条件に一致する全カード集合が 1 件以上ある場合、その結果集合を対象に復習を開始できなければならない
- **FR-002**: システムは、復習開始時に現在の検索条件に整合したカード集合だけを復習対象として確定しなければならない
- **FR-003**: システムは、復習開始後に学習者を復習画面へ遷移させなければならない
- **FR-004**: システムは、復習画面で開始対象となったカードを 1 枚ずつ表示しなければならない
- **FR-005**: システムは、復習画面で次のカードへ進んだとき、開始対象集合の中に含まれる次のカードだけを表示しなければならない
- **FR-006**: システムは、開始対象集合に含まれないカードを、当該復習セッション中に表示してはならない
- **FR-007**: システムは、開始時点の検索結果が 1 件のみであっても、そのカードを対象に復習を開始し完了できなければならない
- **FR-008**: システムは、検索結果が 0 件のとき、復習対象が存在しないことを学習者が理解できる状態を提供しなければならない
- **FR-009**: システムは、検索結果が 0 件のまま復習開始が試みられた場合、空の対象集合で復習を進行させてはならず、条件を見直すか一覧へ戻るための導線を提示しなければならない
- **FR-010**: システムは、復習開始後に一覧画面側の検索条件や表示内容が変更されても、進行中の復習対象集合を自動的に変更してはならない
- **FR-011**: システムは、開始直前に検索結果内の一部カードが利用不能になった場合、利用可能なカードのみを対象に復習を開始し、除外された件数と理由を学習者に分かる形で示さなければならない
- **FR-012**: システムは、開始時点の検索結果件数が 200 件を超える場合、200 件までのカードのみで復習を開始し、超過分の除外件数と理由が上限超過であることを学習者に分かる形で示さなければならない

### Key Entities *(include if feature involves data)*

- **Search Result Set**: 学習カード一覧で開始時点の検索条件に一致する復習候補の集合。検索語や絞り込み条件に一致したカード群を表す
- **Review Session**: 検索結果から開始される復習の単位。開始時点で確定した対象集合と進行状況を持つ
- **Review Target Card**: 復習セッション内で 1 枚ずつ表示されるカード。検索結果集合に含まれていた対象カードを表す

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 学習者の 95% 以上が、検索結果が表示された状態から 15 秒以内に復習を開始できる
- **SC-002**: 検索結果から開始した復習セッションの 100% で、対象外カードが 1 枚も表示されない
- **SC-003**: 検索結果が 2 件以上ある状態で開始した復習セッションの 95% 以上で、学習者が対象カードを順番どおりに最後まで確認できる
- **SC-004**: 検索結果が 0 件の状態では、学習者の 100% が復習を開始できない理由と次の行動を画面上で確認できる
- **SC-005**: 検索結果が 200 件を超える状態で開始した復習セッションの 100% で、開始対象が 200 件に制限されたことと除外件数を学習者が画面上で確認できる
