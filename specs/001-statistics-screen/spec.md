## User Scenarios & Testing *(mandatory)*

### User Story 1 - 学習状況をひと目で把握する (Priority: P1)

ユーザは、統計画面を開いた直後に、自分の学習量と定着状況の全体像を短時間で理解したい。

**Why this priority**: 統計画面の主要価値は現状把握であり、主要指標がすぐ読めないと画面全体の価値が下がるため。

**Independent Test**: 統計画面の初期表示だけを見て、総学習カード数、レビュー完了数、選択中期間内のレビュー回答だけで計算された平均正答率、1日に1回以上レビューした日数の連続で数える連続学習日数を説明できることを確認する。

**Acceptance Scenarios**:

1. **Given** 統計データが存在する, **When** ユーザが統計画面を開く, **Then** 主要 KPI が画面上部でまとめて表示される
2. **Given** 統計画面が表示されている, **When** ユーザが統計サマリーを見る, **Then** 各 KPI が互いに区別できる表示で読める
3. **Given** 前期間比の情報がある, **When** ユーザが KPI を確認する, **Then** 同じ長さの直前期間との比較で現状が改善傾向か停滞傾向か判断できる

---

### User Story 2 - 期間別の変化を見比べる (Priority: P2)

ユーザは、今日、7日間、30日間、全期間を切り替えながら学習傾向の変化を見比べたい。

**Why this priority**: 統計画面は一時点の数値だけでなく推移を見るために使われるため、期間比較は重要な操作である。

**Independent Test**: 期間タブを切り替えたときに、主要 KPI と推移表示が選択期間に応じて更新されることを確認する。

**Acceptance Scenarios**:

1. **Given** 複数期間の統計データが存在する, **When** ユーザが期間タブを切り替える, **Then** 表示中の集計対象期間が明確に分かる
2. **Given** 統計画面が表示されている, **When** ユーザが別の期間を選ぶ, **Then** 主要 KPI と推移表示が同じ構成のまま新しい期間の内容に切り替わる

---

### User Story 3 - 改善ポイントを見つける (Priority: P3)

ユーザは、タグ別の偏りや正答率の弱い領域を見つけて、次に見直す対象を判断したい。

**Why this priority**: 統計は状況確認だけで終わらず、次の学習行動へつながることに価値があるため。

**Independent Test**: 内訳表示とインサイトを見て、改善対象となるタグを 1 つ以上特定できることを確認する。

**Acceptance Scenarios**:

1. **Given** タグ別の統計が存在する, **When** ユーザが内訳セクションを見る, **Then** タグごとの件数と正答率を比較できる
2. **Given** 特定タグの正答率が他より低い, **When** ユーザが統計画面を見る, **Then** 改善が必要な対象が補助説明または強調表示で分かる

### Edge Cases

- レビュー回答が 0 件の場合でも、空状態の案内と次の行動が表示される
- 正答率の履歴など一部データだけ欠損していても、表示可能な統計は維持される
- 統計情報の取得に失敗しても、失敗理由の説明と再試行導線が表示される
- モバイル表示では、統計サマリー、期間切り替え、主要 KPI、推移、内訳、インサイトの順で意味が崩れずに読める
- データなしやエラー状態でも最初のアクションボタンまでキーボードのみで到達できる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST 統計画面をトップレベルナビゲーションから到達できるようにする
- **FR-002**: System MUST 統計画面の上部に統計サマリーを表示する
- **FR-003**: System MUST 主要 KPI として、総学習カード数、レビュー完了数、選択中期間内のレビュー回答だけで計算した平均正答率、1日に1回以上レビューした日数の連続で数える連続学習日数を表示する
- **FR-004**: System MUST 各 KPI に、同じ長さの直前期間と比較した補足情報を表示できるようにする
- **FR-005**: System MUST 期間切り替えとして、少なくとも「今日」「7日間」「30日間」「全期間」を提供する
- **FR-006**: System MUST 初期表示時の既定期間を「7日間」とする
- **FR-007**: System MUST 選択中の期間を視覚的にも文言上も識別できるようにする
- **FR-008**: System MUST 学習量またはレビュー完了数の推移を時系列で理解できる表示領域を提供する
- **FR-009**: System MUST 正答率または定着状況の推移を確認できる表示領域を提供する
- **FR-010**: System MUST タグ別の件数と正答率を比較できる内訳表示を提供する
- **FR-011**: System MUST ユーザが改善対象を判断しやすいインサイトまたは補助説明を表示する
- **FR-012**: System MUST データなし、一部データ欠損、取得失敗の各状態を区別して表示する
- **FR-013**: System MUST レビュー回答が 0 件のとき、データなし状態を表示する
- **FR-014**: System MUST データなし状態で、`/cards/create` への学習カード追加導線と `/review` への復習導線を提示する
- **FR-015**: System MUST 取得失敗時に再試行導線を提示し、最後に取得成功した統計 snapshot がある場合はローカルフォールバックとして再表示できるようにする
- **FR-016**: System MUST デスクトップとモバイルの双方で主要情報が読めるレイアウトにする
- **FR-017**: System MUST キーボード操作のみでも、トップレベルナビゲーション、期間切り替え、主要アクションへ順に移動できるようにする
- **FR-018**: System MUST 統計画面の情報配置と状態バリエーションを ascii_ui.md および ascii_ui.txt と整合させる

### Key Entities *(include if feature involves data)*

- **Statistics Summary**: 総学習カード数、レビュー完了数、平均正答率、連続学習日数などの主要指標をまとめた表示単位
- **Time Range**: 今日、7日間、30日間、全期間など、統計の集計対象期間を表す選択単位
- **Trend View**: 学習量や正答率の変化を時系列で示す表示領域
- **Tag Breakdown**: タグごとの件数と正答率を比較できる内訳表示
- **Statistics State**: 通常表示、データなし、一部データ欠損、取得失敗などの画面状態

### Assumptions

- 統計画面は既存のトップレベルナビゲーションに含まれる独立画面として扱う
- 主要 KPI は画面冒頭で最優先に提示し、その下に推移と内訳を配置する
- 期間切り替えは統計画面全体に作用する共通フィルタとして機能する
- 初期表示の既定期間は 7 日間とする
- 前期間比は、選択中の期間と同じ長さの直前期間を比較対象とする
- 平均正答率は、選択中期間内に行われたレビュー回答だけを母集団として計算する
- 連続学習日数は、1日に1回以上レビューした日を連続日数として数える
- 内訳表示はタグ単位で集計し、詳細分析ではなく次に見直す対象を見つけるための要約表示とする
- データが不足している場合でも、何が表示できて何が表示できないか分かる文言を含める
- データなし状態の主要 CTA は `/cards/create` と `/review` の 2 つを表示する
- 取得失敗時は最後に取得成功した統計 snapshot があればローカル表示を許可し、再試行導線を併記する
- 共有や更新などの補助操作があっても、主要価値は閲覧と判断にある

### Scope Boundaries

- In Scope: 統計画面の情報設計、ASCII UI に基づく画面構成、状態バリエーション、モバイル時の並び順、アクセシビリティ要件
- Out of Scope: 統計計算ロジックの実装、グラフ描画技術、データ保存方式、外部共有機能の仕様詳細

### Dependencies

- 統計画面へのナビゲーション導線が既存 UI に存在すること
- 学習履歴やタグ情報から主要 KPI、推移、内訳を構成できること
- 画面状態ごとの文言と導線が既存のアプリ文体と整合すること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザが統計画面を開いてから 5 秒以内に現在の学習状況を要約して説明できる
- **SC-002**: ユーザの 90% 以上が、表示中の対象期間を誤認せずに識別できる
- **SC-003**: ユーザの 80% 以上が、統計画面だけを見て改善対象のタグを 1 つ以上特定できる
- **SC-004**: データなし、一部データ欠損、取得失敗の各状態で、次に取れる行動を説明できないユーザを発生させない
