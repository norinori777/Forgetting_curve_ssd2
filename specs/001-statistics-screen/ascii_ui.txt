統計画面 ASCII UI

画面の目的:
- 学習状況の全体像を最初の数秒で把握できる
- 期間別の変化を切り替えて確認できる
- 改善が必要な領域を見つけて次の行動につなげられる

状態バリエーション:

1) データなし
+----------------------------------------------------------------------------------+
| まだ統計を表示できるだけの学習履歴がありません                                   |
| まずはカードを追加して復習を始めると、ここに推移と内訳が表示されます              |
| [学習カードを追加]                         [復習画面を見る]                       |
+----------------------------------------------------------------------------------+

2) 一部データ欠損
+----------------------------------------------------------------------------------+
| 一部の統計を表示できません                                                        |
| 正答率の履歴は未取得ですが、学習量とレビュー完了数は確認できます                 |
| [再読み込み]                                                                         |
+----------------------------------------------------------------------------------+

3) 取得失敗
+----------------------------------------------------------------------------------+
| ! 統計情報を取得できませんでした                                                  |
| ネットワークまたはサーバーの状態を確認して、もう一度お試しください                |
| [再試行]                                                                            |
+----------------------------------------------------------------------------------+

モバイル時の意図:
- 上から「統計サマリー」「期間切り替え」「主要 KPI」「学習量の推移」「定着状況」「内訳」の順で縦積み
- KPI は 2 列または 1 列のカードとして崩れずに読める
- 期間切り替えは横並びでも折り返しでも現在選択が明確に分かる
- 主要 CTA は推移の下に残し、スクロール後でも見失いにくくする

アクセシビリティ:
- Tab でトップレベルナビゲーション、期間切り替え、主要 CTA の順に移動できる
- 現在選択中の期間が視覚的にもテキスト的にも識別できる
- データなしとエラー状態でも最初のアクションボタンへすぐ到達できる
